## ОБЩИЕ ПРАВИЛА ДЛЯ ВСЕХ СПРИНТОВ

> Эти правила **обязательны** и имеют приоритет над любыми другими инструкциями.

### Перед началом каждой задачи
1. **Прочитай все указанные файлы** — не начинай писать код пока не прочитал
2. **Проверь что задача ещё актуальна** — возможно предыдущий спринт уже решил проблему
3. **Не угадывай структуру** — только то что видишь в коде

### Во время выполнения
4. **Один файл — одна задача** — не трогай файлы из других задач
5. **Минимальные изменения** — меняй только то что нужно для задачи
6. **Не ломай существующее** — если не уверен что изменение безопасно — остановись и спроси

### После выполнения задачи
7. Напиши: `✅ ЗАДАЧА [N] ВЫПОЛНЕНА`
8. Покажи список изменённых файлов
9. Запусти проверочные команды из задачи

### Запрещено
- ❌ Добавлять новые зависимости без явного указания в задаче
- ❌ Менять схему БД без создания Alembic миграции
- ❌ Оставлять `TODO` или `FIXME` в изменённых файлах
- ❌ Писать `except: pass` без `logger.error()`
- ❌ Хардкодить secrets, токены, пароли

## КОМАНДЫ ФИНАЛЬНОЙ ПРОВЕРКИ ВСЕГО ПРОЕКТА

После завершения всех 4 спринтов:

```bash
# 1. Чистый запуск
python -c "from src.bot.main import main; from src.api.main import app; print('✅ Imports OK')"

# 2. Нет TODO
grep -rn "TODO\|FIXME\|заглушка\|placeholder" src/ \
  --include="*.py" | grep -v "__pycache__"
# Должно быть пусто

# 3. Нет legacy импортов
grep -rn "from src.services" src/ --include="*.py"
# Должно быть пусто

# 4. Тесты
pytest tests/ -v --tb=short --co  # список тестов
pytest tests/ -v                  # запуск

# 5. Линтер
ruff check src/
mypy src/ --ignore-missing-imports  # опционально

# 6. Миграции в порядке
alembic check

# 7. Нет синтаксических ошибок
python -m py_compile $(find src/ -name "*.py" | grep -v __pycache__)
echo "✅ All files compile OK"
```
