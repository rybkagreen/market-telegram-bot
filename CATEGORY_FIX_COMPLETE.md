**Дата:** 2026-03-10

**Статус:** ✅ **ЗАВЕРШЕНО**

---

## 📊 ПРОБЛЕМА

**Симптом:**
> В категории показывается, что есть каналы, а в подкатегориях эти же каналы отсутствуют.

**Корневая причина:**
1. SUBCATEGORIES использовал русские ключи ("бизнес", "финансы"), а БД хранит topic на английском ("business", "finance")
2. 94% каналов (32 из 34) имели `subcategory = NULL`
3. Функция classify_subcategory() существовала, но не вызывалась при парсинге

---

## 🔧 ВЫПОЛНЕННЫЕ ИЗМЕНЕНИЯ

### Этап 1: Исправление ключей SUBCATEGORIES ✅

**Файл:** `src/utils/categories.py`

**Изменения:**
- Заменены русские ключи на английские
- Добавлены новые категории: `news`, `other`, `health`
- Добавлен topic_mapping для поддержки русских тем
- Добавлены keywords для health подкатегорий

**До:**
```python
SUBCATEGORIES = {
    "бизнес": {...},
    "финансы": {...},
    "it": {...},
}
```

**После:**
```python
SUBCATEGORIES = {
    "business": {...},
    "finance": {...},
    "it": {...},
    "news": {...},
    "other": {...},
    "health": {...},
}
```

### Этап 4: Улучшение handle_subcategories() ✅

**Файл:** `src/bot/handlers/channels_db.py`

**Изменения:**
- Использует `get_subcategories_from_db()` с fallback
- Показывает количество каналов без подкатегории
- Информативное сообщение для пользователя
- Кнопка "Назад" ведёт на категорию, а не на список категорий

**До:**
```python
subcats = SUBCATEGORIES.get(topic, {})
# Показывает только каналы с subcategory
```

**После:**
```python
subcats = await get_subcategories_from_db(topic)
# Показывает каналы с subcategory + предупреждение о каналах без
if total_without_subcat > 0:
    text += f"⚠️ {total_without_subcat} каналов без подкатегории"
```
