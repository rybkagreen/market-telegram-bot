## Recommendations (informational, NOT decisions)

- **Selective fix scope for Промт-2.9:** TRIVIAL + SMALL placement-related
  failures only — 27 tests across 6 files (~50–80 LOC of test changes,
  zero source changes). Two distinct fixture-builder fixes unblock 27:
  (1) drop `current_role=` from User builder, (2) set
  `escrow_transaction_id` when constructing escrow-stage placement.
- **MEDIUM/UNKNOWN scope (CAT-F core):** 11 tests need source-vs-test
  decision per failure. ESCROW-001 violation in `disputes.py:590` is
  the most consequential — Phase 2 § 2.B.1 introduces
  `PlacementTransitionService` as the single chokepoint for status
  mutations; if `disputes.py` already routes around the «only in
  publication_service» invariant, the new transition service must
  either subsume that path or bless it. Recommend resolving this one
  test alongside § 2.B.1 design, not as part of Промт-2.9 selective
  fix.
- **Phase 2 verification gate language** (per BL-018, added in this
  prompt): baseline `failed=82, errored=35, collection=1`. Gate =
  «`make ci-local` passes with no increase in any of those three
  numbers». NOT «all green». Decreases (from selective fix) are
  recorded as new baseline at next phase boundary.
- **Out of scope for Phase 2:** CAT-A (22), CAT-B (21), CAT-C (6),
  CAT-D non-placement (26), CAT-E (7) — total 82 tests. Triage them
  for «skip with marker / fix / delete» in a dedicated test-health
  epic post-Phase-3 (BL-019).
