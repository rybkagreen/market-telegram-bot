- **A.3 — Payout entity mismatch.** `PayoutRequest` lifecycle (`pending → processing → paid` etc.) is owner-aggregate, not per-placement. Plan §3.B.1 phrasing "completed → payout_processing" maps awkwardly: there is no `payout_processing` placement state (terminal at `completed`). Per-placement payout gates evaluate against the placement, not the payout — the conceptual boundary stays with the future `PayoutComplianceService`.
- **A.4 — Acts entity.** `Act` auto-generated by `publication_service.delete_published_post:432-441` after `placement.deleted_at` is set. Generation is wrapped in try/except (line 438-441 — silent-fail by design). G14 catches the silent-failure case at gate-time. `ActRepository.get_by_placement_request` returns the single first row (Acts are 1:1 per placement; `ActService` early-exits if existing).
- **A.5 — G13 signal choice.** `placement.deleted_at IS NOT NULL` chosen over `scheduled_delete_at <= now()` and `published_at + format_duration <= now()` because it pairs with G14's trigger condition (`Act` requires `deleted_at IS NOT NULL`) and avoids Celery scheduling-lag races where G13 could pass before `deleted_at` is set while G14 still fails.
- **A.10 — 5b.1 attribution conflict.** 5b.1 closure CHANGES (`CHANGES_2026-05-02_phase3b-5b1-schema-additives.md`) attributes G06/G16/G17/G18 **bodies** to Phase 5 (Payout Provider Implementation). 5b.6 ships **markers** for G17/G18, not real bodies — markers reserve the namespace, complete Phase 3b body coverage, and surface the phase boundary to API consumers. Phase 5 swap = marker body → real body, no contract change. Two attributions are complementary, not conflicting.
- **A.13 — Scope split recommendation.** Single sub-block 5b.6 = all 4 gates (G13+G14 real + G17+G18 markers). Mirrors 5b.4 precedent (G04/G05 real + G06 marker bundled in one file).
- **CC.5 — Race conditions.** `PayoutService.approve_request`/`reject_request` use `with_for_update` row lock on `PayoutRequest`. G13/G14 read `placement.deleted_at` and `Act.placement_request_id` — frozen post-completion; no race risk for 5b.6 reads.
